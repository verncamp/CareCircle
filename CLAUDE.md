# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

CareCircle is an iPhone-first SwiftUI app for adult children coordinating care for an aging parent: appointments, tasks, a document vault, family roles, shared finances, an emergency PDF packet, HealthKit vitals, and on-device AI. The core app is **local-first and private** — no account server, and no backend dependency. iOS 26+ / Xcode 26+.

## Build, run, test

The Xcode project is generated by **XcodeGen** from `project.yml`. `CareCircle.xcodeproj` is committed, but treat `project.yml` as the source of truth: after adding/removing/renaming files or changing build settings, regenerate.

```bash
brew install xcodegen          # one-time
xcodegen generate              # regenerate the .xcodeproj after editing project.yml or adding files

# Build (simulator)
xcodebuild -scheme CareCircle -destination 'platform=iOS Simulator,name=iPhone 16 Pro' build

# Run all tests
xcodebuild test -scheme CareCircle -destination 'platform=iOS Simulator,name=iPhone 16 Pro'

# Run a single test class or method
xcodebuild test -scheme CareCircle \
  -destination 'platform=iOS Simulator,name=iPhone 16 Pro' \
  -only-testing:CareCircleTests/EmergencyPacketTests
# ...append /testMethodName to target a single test
```

Sources are added by directory in `project.yml` (`Models`, `Views`, `Helpers`, plus the two top-level Swift files), so new files in those folders are picked up on the next `xcodegen generate` without editing the manifest.

## Architecture

### Routing
`CareCircleApp` builds the `ModelContainer` and hosts `AppRouter` (`ContentView.swift`). Routing is driven entirely by the `@AppStorage("appMode")` string:
- `"none"` → `WelcomeView` (Get Started / Try Demo)
- `"signup"` → `OnboardingView`
- `"real"` → `OnboardingView` if no `ParentProfile` exists yet, else `MainContentView`
- `"demo"` → `MainContentView(isDemo: true)`

`MainContentView` switches on `horizontalSizeClass`: `TabView` on iPhone (compact), `NavigationSplitView` sidebar on iPad (regular). The six tabs are the `AppTab` enum.

### Persistence (SwiftData)
The schema is declared once in `CareCircleApp.init`. `ModelConfiguration` is chosen by `appMode`:
- **demo** → `isStoredInMemoryOnly: true` (nothing persists; cleared on exit)
- **everything else** → on-device persistent store, `cloudKitDatabase: .none` (CloudKit was intentionally removed — `Helpers/CloudKitAccountManager.swift` is deleted)

If the container fails to init (e.g. a schema change with no migration), it deletes the store files and retries once — so **breaking model changes wipe local data** rather than crash. Keep that in mind when editing `Models/`.

`ParentProfile` is the root entity; Appointments, Tasks, Documents, FamilyMembers, Expenses, and UpdateFeedItems cascade-delete from it. Relationship inverses are declared explicitly to make cascade deletion reliable. `HealthMetricSnapshot` is not persisted (HealthKit is read live).

### On-device AI
`Helpers/AIAssistant.swift` wraps Apple `FoundationModels` for daily briefings, note summaries, task extraction, and document categorization. Always gate on `AIAvailabilityState` (`SystemLanguageModel.Availability`) — most devices/simulators report unavailable, and **all core workflows must function with AI off**. Devices ineligible for Apple Intelligence may optionally use a cloud summary endpoint, gated behind `@AppStorage("cloudSummariesEnabled")` (off by default) and the `INFOPLIST_KEY_CARECIRCLE_SUMMARY_API_URL` build setting.

### Other helpers
- `HealthKitManager` — read-only vitals (heart rate, SpO₂, steps, BP, HRV, 7-day chart).
- `NotificationManager` — local `UNUserNotificationCenter` scheduling; all defaults keys live under `NotificationManager.Keys` (reference these constants, don't hardcode AppStorage strings).
- `EmergencyPacketGenerator` — Core Graphics PDF; `OCRHelper` — Vision OCR for scans; `DocumentScannerView` — VisionKit.
- `RegionProfile` / `RegionalTemplateSeeder` — US/Canada document template sets seeded during onboarding.
- `SampleDataGenerator` — demo seed data; also `clearAllData(modelContext:)` when leaving demo mode.
- `ActivityFeedHelper` — logs `UpdateFeedItem` entries for care actions.

## Conventions

- **`asyncRun { }` instead of `Task { }`** (defined in `Theme.swift`). The SwiftData model is named `Task`, which shadows Swift concurrency's `Task`. `asyncRun` has `@Sendable` and `@MainActor` overloads. When you must reference the concurrency type directly, use `_Concurrency.Task`.
- **Saving:** call `modelContext.safeSave()` (in `SaveHelper.swift`) — it returns a `Bool` instead of throwing. Pair with `.saveErrorBanner(show:)` on the view for user-visible failure feedback.
- **Design system in `Theme.swift`:** use `.glassCard()` for cards (`.ultraThinMaterial`) and `.screenBackground()` for the Hearth clay/sage/apricot wash. App tint is `.careTint`.
- New code should keep the local-first, no-network posture. Don't reintroduce CloudKit or add network calls outside the opt-in cloud-summary path.

## Tests

`CareCircleTests/` (unit, ~8 files) covers models, cascade deletion, expense accounts, form validation, onboarding, regional documents, emergency packets, and the activity feed. `CareCircleUITests/` covers navigation and core flows.
